## About

This module introduces Retrieval-Augmented Generation (RAG) as a practical way to ground LLM outputs in real evidence. Participants will explore how RAG compares to fine-tuning, walk through its core building blocks, and use a simple mental model to understand when and why it matters for compliance and production systems.

## Module Objectives

- Explain what Retrieval-Augmented Generation (RAG) is and how it differs from fine-tuning.
- Identify and describe the six core RAG primitives (documents, chunking, embeddings, vector store, retriever, generator).
- Build and evaluate a simple RAG pipeline, applying validation and monitoring techniques to ensure accuracy and reliability.
